Assignment 3 - HTML Page WebDAV

In 1989, Tim Berners-Lee created the World Wide Web and the web browser called WorldWideWeb which was renamed Nexus to avoid confusion, but ended up discontinued in 1994. It could view AND edit webpages and was the first WYSIWYG (what you see is what you get) HTML editor.



Eventually browsers became read-only as web browsers grew in popularity. There were risks of corruption and would need highly skilled developers monitoring. A group of people wanted to bring it back, and so Jim Whitehead organized a meeting with members of the World Wide Web Consortium in 1996 to discuss the problem of authoring on the web and created the Internet Engineering Task Force (IETF) which is “an open standards organization, which develops and promotes voluntary Internet standards, in particular the standards that comprise the Internet protocol suite (TCP/IP).” And eventually webDAV was created.

The advantages of using webDAV in your business is the ability to edit files directly from the server directly from your computer. This is good because developers from all over the world are able to work on the same project together (called collaborative authoring.) This saves time and allows for more work to be done on a project rather than having 1 person or a small team working on it from one location. Access to files requires a username and password.

Other advantages include

  • Operating system integration
  • Free to use
  • Close integration with web services
  • Version Control
  • Transport encryption
  • Remote access
  • Centralized storage
  • Version Control
  • File locking

I have had personal experience with systems like this during my time as a graphic designer. In first job in advertising, when in the office we used a shared server that we could access as long as we were on the correct wifi, however if we were to leave the building, we would then have to login from a different site in order to access files.

Today, I use GoogleDrive and Dropbox daily for collaborative work. During meetings we all open Google Sheets to view information and watch real-time edits applied to the document. Other companies that use webDAV are Apache HTTP Server, Microsoft IIS, Box.com, WordPress, Drupal, Microsoft Sharepoint, Subversion, Git, Windows Explorer, macOS Finder, Microsoft Office, Apple iWork, and Adobe.

Alternatives today to webDAV are

  • FTP
  • SSH
  • Wikis
  • Distributed filesystems
  • Cloud file storage
